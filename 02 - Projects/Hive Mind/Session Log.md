# Hive Mind — Session Log (Append-Only)

> Every Claude Code session — in any workspace — writes a 1–5 line entry here on close via the `hive-mind-sync` skill. Entries are reverse-chronological. This file is the single thread that connects all project workspaces into one continuous conversation for future agents.

## How to write an entry

Keep entries tight. Format:

```
### YYYY-MM-DD — Workspace name — one-line headline
- What was done (1–3 lines, outcomes not process)
- Cross-project impact, if any (mentions of other workspaces)
- Canonical facts promoted to Unity: [[wiki-links to updated notes]]
```

## What belongs here (vs. project-local memory)

**Write here** when the work touches more than one workspace, produces a canonical fact (client/server/credential), or would be useful to an agent in a DIFFERENT project next week.

**Keep in project-local memory** when the work is only about that codebase's internals — a fix to a specific file, a local test failure, a refactor inside one module. Those stay in `~/.claude/projects/<slug>/memory/`.

### 2026-05-03 (Suleman Remittance v10.0.39 → v10.0.40 — IAS 21 FX gain/loss fix on cross-currency invoice payment) — D:/Development — solves the deferred partial-paid + EUR Bank -2,700 bug
- **Trigger:** user asked me to act as expert accountant and solve the deferred FX-residual bug from v10.0.27.
- **Root cause:** v10.0.27 `_create_odoo_payment` (step-6 hawala JE) wrote `debit = self.total` (foreign EUR numerical value) while setting `currency_id = EUR` but NOT populating `amount_currency`. Odoo's invariant is that `debit`/`credit` are ALWAYS in COMPANY currency; the foreign-currency leg lives in `amount_currency` (signed). Without `amount_currency`, the foreign leg defaults to 0. On Daleel (USD-functional) processing 15k EUR: step-4 invoice's AR was @ 17,647.06 USD / +15,000 EUR; step-6 cleared AR @ 15,000 USD / 0 EUR (treating foreign as company). `reconcile()` saw foreign mismatched and couldn't trigger Odoo's exchange-move generation, leaving 2,647.06 USD residual → `payment_state='partial'`. Same gap manifested as -2,700 EUR phantom on Bank dashboard.
- **Fix:** rewrote builder using `foreign_ccy._convert(amount, company_ccy, company, date_ref)` for proper company-currency conversion + `amount_currency` populated (signed) on every line. `reconcile()` now auto-creates the FX exchange move per **IAS 21 §28** (exchange differences arising on settlement at rates different from initial recognition shall be recognised in P&L) — posts to `company.income_currency_exchange_account_id` (gain, acct id 27) or `expense_currency_exchange_account_id` (loss, acct id 38). Both pre-configured on Daleel during initial COA install.
- GitHub `engrmuhammadamirnazir/remittance_management` `main 184fd59 → 9d099f6` + tag v19.0.10.0.40 (NO AI attribution per client-repo rule). Both DBs at `installed | 19.0.10.0.40` HTTPS 303.
